    # 盒子模型

    页面上由一个个盒子模型组成一个大的盒子

    盒子由外及内是：margin border padding content

    total-width=margin+border+padding+width

    total-height=margin+border+padding+height

    标准盒子：

    width=content-width

    total-width=margin+border+padding+width

    total-height=margin+border+padding+height

    IE怪异盒子

    width=content-width+padding+border

    total-width=margin+width

    total-height=margin+height

    给div设置宽高是设置的width和height

    如果给div设置同样的width height和padding、border

    那么content box> border box，

    div {
  width: 160px;
  height: 80px;
  padding: 20px;
  border: 8px solid red;
  background: yellow;
}

.content-box {
  box-sizing: content-box;
  /* Total width: 160px + (2 * 20px) + (2 * 8px) = 216px
     Total height: 80px + (2 * 20px) + (2 * 8px) = 136px
     Content box width: 160px
     Content box height: 80px */
}

.border-box {
  box-sizing: border-box;
  /* Total width: 160px
     Total height: 80px
     Content box width: 160px - (2 * 20px) - (2 * 8px) = 104px
     Content box height: 80px - (2 * 20px) - (2 * 8px) = 24px */
}

    # 定位

    ##标准流定位

    ###标准流

    默认情况下元素都是按照标准流从左到右从上倒下排布的

    ###脱标元素

    可以随意设置宽高，不设置默认由content决定，不受标准流约束，父元素不再获得脱标元素的宽高，可能发生高度坍塌的情况(父元素没有高度了)

    ###margin padding定位

    缺点是：

    设置一个与元素的margin或者padding，会影响到标准流中的其他元素的定位效果，

    不便于实现元素层叠效果

    ##position定位

    position:static|relative|absolute|fixed

    可设置正负值

    # static:静态定位

    ​ 默认值，按照normal flow定位，设置上下左右无用

    ###position:fixed;

    元素脱离标准流

    定位参照对象：视口

    可设置上下左右

    适用场景：页面滚动，元素固定不动

    # relative：相对定位

    ​ 在标准流，设置上下左右，相对于自己原先的位置定位，原先行内元素，设置相对定位，再设置上下左右移懂

    ###position:absolute;

    脱离标准流

    定位参照对象：最近的上一级元素，如果没有则是视口

    可设置上下左右

    ###子绝父相

    子元素相对于父元素定位，父元素不脱标

    ###positon取值总结

    ###绝对定位技巧

    居中

    根据公式

    设置绝对定位元素覆盖相对元素(父元素)

    设置子元素在父元素中水平垂直居中

    # VW VH

    vw 和 vh 分别是相对于屏幕视口宽度和高度而言的长度单位：

    • 1vw = 视口宽度均分成 100 份中 1 份的长度；
    • 1vh = 视口高度均分成 100 份中 1 份的长度；

    在 JS 中 100vw = window.innerWidth，100vh = window.innerHeight。

    # 媒体查询

    媒体查询是指针对不同的设备、特定的设备特征或者参数进行定制化的修改网站的样式。

    外部多样式文件引入

    你可以通过给 `` 加上 media 属性来指定该样式文件只能对什么设备生效，不指定的话默认是 all，即对所有设备都生效：

    <link rel="stylesheet" src="styles.css" media="screen" />
<link rel="stylesheet" src="styles.css" media="print" />
复制代码

    都支持哪些设备类型？

    • all：适用于所有设备；
    • print：适用于在打印预览模式下在屏幕上查看的分页材料和文档；
    • screen：主要用于屏幕；
    • speech：主要用于语音合成器。

    需要注意的是：通过 media 指定的 资源尽管不匹配它的设备类型，但是浏览器依然会加载它。

    # CSS自定义属性（变量）

    通常定义在根伪类 :root

    大小写敏感

    :root {
  --main-bg-color: brown;
}

    使用一个局部变量时用 var()函数包裹

    element {
  background-color: var(--main-bg-color);
}

    注意1

    自定义属性会继承。这意味着如果在一个给定的元素上，没有为这个自定义属性设置值，在其父元素上的值会被使用。

    <div class="one">
  <div class="two">
    <div class="three"></div>
    <div class="four"></div>
  </div>
</div>

    .two {
  --test: 10px;
}

.three {
  --test: 2em;
}

    在这个情况下， var(--test) 的结果分别是：

    • 对于元素 class="two" ：10px
    • 对于元素 class="three" ：2em
    • 对于元素 class="four" ：10px （继承自父属性.two）
    • 对于元素 class="one" ：非法值，会变成自定义属性的默认值

    注意2

    使用css变量，可以说设置备用属性值

    var函数的第一个参数是自定义属性的名称。如果提供了第二个参数，则表示备用值，当自定义属性值无效时生效。第二个参数可以嵌套，但是不能继续平铺展开下去了(出现第三个参数)

    # BFC

    https://developer.mozilla.org/zh-CN/docs/Web/Guide/CSS/Block_formatting_context

    BFC 是 Block Formatting Context的缩写，即块级格式化上下文。BFC是CSS布局的一个概念，是一个独立的渲染区域，规定了内部box如何布局， 并且这个区域的子元素不会影响到外面的元素

    # 开启BFC

    • 根元素
    • overflow:hidden|auto|scroll
    • 浮动元素 float:left|right
    • 绝对定位元素position:absolute|fixed
    • 行内块元素display:inline-block
    • 弹性元素display:flex
    • 网格元素display:grid
    • 表格单元格（元素的 display (opens new window) 为 table-cell，HTML表格单元格默认为该值）
    • 表格标题（元素的 display (opens new window) 为 table-caption，HTML表格标题默认为该值）
    • 匿名表格单元格元素（元素的 display (opens new window) 为 table、``table-row、 table-row-group、``table-header-group、``table-footer-group（分别是HTML table、row、tbody、thead、tfoot 的默认属性）或 inline-table）
    • contain (opens new window) 值为 layout、content或 paint 的元素
    • 多列容器（元素的 column-count (opens new window) 或 column-width (en-US) (opens new window) 不为 auto，包括 ``column-count 为 1）
    • column-span 为 all 的元素始终会创建一个新的BFC，即使该元素没有包裹在一个多列容器中（标准变更 (opens new window)，Chrome bug (opens new window)）。

    # BFC规则

    页面中的一块渲染区域，块级元素独占一行，依次垂直排列

    BFC区域不与float box重叠

    内部的块级元素之间的距离由margin决定，一个BFC内的boxmargin会重叠， 解决：单独创建BFC

    BFC内的元素不影响外部元素

    计算BFC高度时，浮动元素参与计算

    # 使用场景

    # 元素隐藏方法

    1.opacity：0，该元素隐藏起来了，但不会改变页面布局，并且，如果该元素已经绑定 一些事件，如click 事件，那么点击该区域，也能触发点击事件的

    2.visibility：hidden，该元素隐藏起来了，但不会改变页面布局，但是不会触发该元素已 经绑定的事件 ，隐藏对应元素，在文档布局中仍保留原来的空间（重绘）

    3.display：none，把元素隐藏起来，并且会改变页面布局，可以理解成在页面中把该元素。 不显示对应的元素，在文档布局中不再分配空间（回流+重绘）

    # 响应式

    页面的开发与设计应该根据用户的行为和设备环境（系统平台、屏幕尺寸等）进行相应的响应和调整，

    同一个网站在不同的终端有不同的显示效果，能够兼容各个终端，不需要为每个终端设计一个版本

    设置viewport／scale，使用流体宽度（设置max／min-width，不使用固定宽度），使用媒体查询，使用相对单位rem转换，移动优先

    响应式+rem

    给根元素设置font-size:10px;

    子元素font-size:2rem; //20px

    # H5C3新特性

    语义化标签

    <header>	<nav> <aside>	<section>	<footer> <article>

    1. 概念：

      HTML5的语义化指的是合理正确的使用语义化的标签来创建页面结构。【正确的标签做正确的事】

    2. 语义化标签：

      header nav main article section aside footer

    3. 语义化的优点:

      • 在没CSS样式的情况下，页面整体也会呈现很好的结构效果
      • 代码结构清晰，易于阅读，
      • 利于开发和维护 方便其他设备解析（如屏幕阅读器）根据语义渲染网页。
      • 有利于搜索引擎优化（SEO），搜索引擎爬虫会根据不同的标签来赋予不同的权重
